# Draft: 项目完善功能规划

## 项目概况

**Ink & Vellum (ai-novel-agent)** — 一个基于 Vite + React + TipTap v3 + Tauri + Tailwind v4 的小说创作工作室。

### 已有的核心能力
- 项目管理（建/删/列项目）
- 章节 CRUD + 状态机（draft/writing/done/paused）
- TipTap 富文本编辑器 + 幽灵文字 + / 命令菜单 + 选中悬浮栏
- AI 聊天面板（@提及 + RAG 引用 + 三位置插入 + 历史管理）
- ContextBus + AIService 编排层（12 个 AI 命令）
- LLM 多提供商支持（OpenAI/Anthropic/Ollama/DeepSeek/自定义）
- RAG 引擎（LanceDB 向量检索 + 关键词回退）
- 写作灵魂系统（SKILL-*.md 文件导入 + 身份卡/模型引用/工作流）
- 6 个内置写作人格 + AI 自动生成人格
- 写作风格配置（语气/视角/温度）
- 导出（章节/项目 → Markdown/TXT/HTML）
- 章节搜索
- 全局大纲 + 多层级大纲 + 拖拽排序
- 章节笔记面板 + AI 辅助
- 完成的设计系统、Toast、ErrorBoundary
- 22 个测试文件，250 个测试

### 发现的缺口和改进点

#### 🔴 P0 — 致命问题
1. **SQLite 纯内存运行**：`new SQL.Database()` 不持久化，每次刷新页面数据全丢。`saveDatabase()`/`loadDatabase()` 存在但从未被调用

#### 🟠 P1 — 重大缺失
2. **角色/世界观无 UI**：数据库有 `characters` 和 `world_settings` 表，RAG 引擎能索引搜索，但完全没有用户界面来创建/编辑/管理
3. **写作人格/SKILL 选择无 UI**：创建项目时可以选，但进项目页后没有切换 UI
4. **编辑器工具栏太基础**：只有 B/I/H1/H2/H3，缺少撤销/重做/列表/引用/代码块/对齐等

#### 🟡 P2 — 体验提升
5. **无键盘快捷键体系**：只有 Cmd+S，缺少 Cmd+B/I/Z/Shift+Z/F 查找等
6. **无搜索对话框集成**：`SearchDialog.tsx` 存在但未集成到 UI
7. **无统计看板**：项目级总字数、章节进度、写作频率等
8. **无字数目标追踪 UI**：wordGoal 存在但无进度追踪仪表盘
9. **无响应式布局**：仅桌面端，无法在小屏使用
10. **无首次引导/新手教程**

#### 🟢 P3 — 锦上添花
11. **暗/亮主题切换**（已有完整 dark 主题但无 light 主题）
12. **写作日历/统计**（热力图、连续写作天数）
13. **版本历史/草稿管理**（每章节可回退到历史版本）
14. **本地图片上传/管理**
15. **AI 写作建议历史**（幽灵文字结果可保存查看）
16. **更多导出格式**（PDF, EPUB）
